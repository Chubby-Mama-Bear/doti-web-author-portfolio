# 2026-04-21 — Intake Workflow Design & Governance Structuring

## Phase
Workflow Design — Intake Standardization

---

## Context

Team discussions identified recurring issues with inconsistent web update requests, unclear submission ownership, and lack of structured intake validation.

Initial communication workflows were informal and varied across requestors.

---

## Key Contributions

### Standardized Shared Mailbox Intake Messaging

Configured a standardized shared mailbox signature and automatic reply template defining:

- Expected response time (2–5 business days)
- Required submission details
- Structured request formatting expectations

This established the first consistent intake messaging layer across request channels.

### Designed Structured Request Intake Workflow

Developed a proposed intake workflow model including:

- Submission requirements
- Supervisor authorization consideration
- Role-based request routing
- Pre-publication verification workflow
- Standardized follow-up communication structure

This proposal created the foundation for repeatable request handling.

### Evaluated Platform Form Options

Reviewed available intake tooling options:

- OpenForms (advanced workflow platform)
- Legacy Forms (simplified structured intake tool)

Recommended Legacy Forms as an initial implementation strategy based on maintainability and operational simplicity.

### Introduced Governance Authorization Model

Proposed restricting intake submissions to supervisor-level or designated contacts to:

- Prevent unauthorized changes
- Improve accountability
- Reduce redundant requests

This initiated governance-level workflow controls.

(Source: :contentReference[oaicite:3]{index=3})

---

### Implemented Pre-Publish Verification Step

Designed a validation checkpoint requiring:

- Exported webpage preview (PDF or screenshot)
- Requestor confirmation prior to publication

This reduced risk of publishing incorrect updates.

(Source: :contentReference[oaicite:4]{index=4})

---

### Defined Responsibility Routing Model

Proposed role-based assignment structure:

- Survey pages → Mike
- SUDP pages → Chloe
- Other requests → First-seen assignment basis

This clarified ownership boundaries across teams.

## Outcome

Established a structured intake and governance framework supporting:

- Standardized request communication
- Role clarity
- Accountability controls
- Reduced ambiguity in update workflows

This day represents the transition from informal intake handling to structured operational design.
