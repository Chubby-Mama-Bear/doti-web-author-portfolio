# Website Update Submission Format Template

## Purpose

To standardize how departments submit website update requests, ensuring that required information is complete, consistent, and easy to process.

This template supports structured intake and reduces delays caused by incomplete or unclear requests.

---

## Problem Observed

Without a structured submission format:

- Requests arrived with missing details
- Follow-up clarification was frequently required
- Processing time increased
- Work tracking became inconsistent
- Communication overhead increased

---

## Proposed Submission Format

Departments submitting update requests should include the following information:

---

### Required Fields

**Department Name**

Name of the submitting department.

---

**Requested Update Type**

Examples:

- Address Update
- Document Update
- Content Correction
- Other

---

**Page or Document Location**

Provide:

- Website URL  
OR  
- Document Name

---

**Description of Requested Change**

Clearly describe:

- What needs to change
- Where the change occurs
- What the new content should be

---

**Supporting Materials**

Attach:

- Updated document  
OR  
- Screenshot showing change location

---

**Requested Completion Timeline**

If applicable, specify urgency level:

- Standard  
- Priority  
- Time-sensitive

---

## Example Submission

Department Name: ROWS

Update Type: Address Update

Location: ROWS Permit Application PDF

Requested Change:  
Replace mailing address on page 2 with updated department mailing location.

Supporting Materials:  
Updated PDF attached

Timeline: Standard
